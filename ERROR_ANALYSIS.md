# MP4ビデオファイル「bad encoding」エラー原因解析

## エラー情報

- **エラーコード**: 3 (INVALID_ARGUMENT)
- **エラーメッセージ**: `Invalid recognition 'config': bad encoding..`
- **発生ファイル**: MP4ビデオファイル（296.04 MB）
- **設定**: `languageCode: 'ja-JP'`のみ（最小限の設定）

## 現在の実装

1. **GCS URIを使用**: `gs://doya-interview-storage/...`
2. **encodingは設定しない**: APIが自動検出するように設定
3. **最小限の設定**: MP4ビデオファイルの場合、`languageCode`のみ
4. **longRunningRecognizeを使用**: ビデオファイルの場合

## 考えられる原因

### 1. MP4ファイル内の音声コーデックがサポートされていない

**問題点**:
- MP4はコンテナ形式で、内部の音声コーデックが様々（AAC、MP3、AC3など）
- Google Cloud Speech-to-Text APIが音声トラックを抽出できない可能性
- 特に、MP4ファイル内の音声がAACコーデックでエンコードされている場合、APIが認識できない可能性

**検証方法**:
- ファイルの音声コーデックを確認（FFprobeなど）
- サポートされているコーデック（FLAC、LINEAR16など）に変換してテスト

### 2. Google Cloud Speech-to-Text APIのMP4サポート制限

**問題点**:
- APIがMP4ファイルを直接サポートしていない可能性
- MP4ファイルから音声を抽出する機能が限定的

**検証方法**:
- 公式ドキュメントでMP4ファイルのサポート状況を確認
- 別のMP4ファイルでテスト

### 3. ファイルサイズまたはファイル形式の問題

**問題点**:
- 296MBという大きなファイルサイズ
- MP4ファイルのバージョンやエンコーディング方式

**検証方法**:
- より小さいMP4ファイルでテスト
- 別のMP4ファイルでテスト

## 推奨される解決策

### 解決策1: 音声ファイルに変換（推奨）

MP4ファイルから音声を抽出し、サポートされている形式（FLAC、LINEAR16など）に変換する。

**メリット**:
- APIが確実に処理できる形式を使用
- エンコーディングエラーを回避

**デメリット**:
- 変換処理が必要
- 追加のストレージ容量が必要

### 解決策2: 別のAPIサービスを使用

MP4ファイルを直接処理できる別のサービスを使用する。

**例**:
- Google Cloud Video Intelligence API（動画解析）
- 別の文字起こしサービス

### 解決策3: クライアント側で音声抽出

アップロード時にクライアント側で音声を抽出し、音声ファイルとしてアップロードする。

## 次のステップ

1. **エラーログを確認**: 実際に送信されているリクエスト設定を確認
2. **ファイル形式を確認**: MP4ファイル内の音声コーデックを確認
3. **テスト**: 別のMP4ファイルまたは音声ファイルでテスト
4. **解決策の実装**: 上記の解決策のいずれかを実装
