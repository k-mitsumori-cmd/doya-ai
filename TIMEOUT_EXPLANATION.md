# 構成案生成APIのタイムアウト時間について

## タイムアウト時間の計算方法

構成案生成APIでは、文字起こしが完了するまで待機しますが、タイムアウト時間はファイルサイズに基づいて動的に計算されます。

### 基本設定

- **最小待機時間**: 5分
- **最大待機時間**: 3日（72時間）
- **バッファ時間**: 処理時間の50%、最低10分

### 計算式

#### 動画ファイルの場合

1. **音声長の推定**: `1GB ≈ 1時間`（一般的な1080p動画）
2. **音声抽出時間**: `1分の動画 ≈ 約4秒`（FFmpeg処理）
3. **文字起こし時間**: `音声長の20%`（リアルタイムファクター）
4. **バッファ時間**: `処理時間の50%、最低10分`
5. **合計**: `(抽出時間 + 文字起こし時間) + バッファ時間`

#### 音声ファイルの場合

1. **音声長の推定**: `1MB ≈ 約1.08分`（MP3 128kbps）
2. **文字起こし時間**: `音声長の20%`（リアルタイムファクター）
3. **バッファ時間**: `処理時間の50%、最低10分`
4. **合計**: `文字起こし時間 + バッファ時間`

### 計算例

#### 1GBの動画ファイル

- 音声長: 1時間（60分）
- 音声抽出時間: 60分 × 4秒 = 240秒 = 4分
- 文字起こし時間: 60分 × 60秒 × 0.2 = 720秒 = 12分
- 処理時間: 4分 + 12分 = 16分
- バッファ時間: max(10分, 16分 × 0.5) = 10分
- **合計タイムアウト時間**: 16分 + 10分 = **26分**

#### 10GBの動画ファイル

- 音声長: 10時間（600分）
- 音声抽出時間: 600分 × 4秒 = 2400秒 = 40分
- 文字起こし時間: 600分 × 60秒 × 0.2 = 7200秒 = 120分（2時間）
- 処理時間: 40分 + 120分 = 160分
- バッファ時間: max(10分, 160分 × 0.5) = 80分
- **合計タイムアウト時間**: 160分 + 80分 = **240分（4時間）**

#### 1GBの音声ファイル

- 音声長: 1GB × 1024MB × 1.08分 = 1106分（約18.4時間）
- 文字起こし時間: 1106分 × 60秒 × 0.2 = 13272秒 = 221分（約3.7時間）
- バッファ時間: max(10分, 221分 × 0.5) = 111分
- **合計タイムアウト時間**: 221分 + 111分 = **332分（約5.5時間）**

#### 10GBの音声ファイル

- 音声長: 10GB × 1024MB × 1.08分 = 11059分（約184時間）
- 文字起こし時間: 11059分 × 60秒 × 0.2 = 132708秒 = 2212分（約36.9時間）
- バッファ時間: max(10分, 2212分 × 0.5) = 1106分
- 計算上の合計: 2212分 + 1106分 = 3318分（約55.3時間）
- **実際のタイムアウト時間**: **3日（72時間）**（最大待機時間の上限）

### 待機間隔

ファイルサイズに応じて、状態確認の間隔が調整されます：

- **1時間未満**: 2秒間隔
- **1時間以上6時間未満**: 5秒間隔
- **6時間以上24時間未満**: 10秒間隔
- **24時間以上**: 30秒間隔

### まとめ

- **最小タイムアウト**: 5分
- **最大タイムアウト**: 3日（72時間）
- **ファイルサイズが大きいほど、タイムアウト時間も長くなる**
- **動画ファイルは音声ファイルより処理が速いため、タイムアウト時間も短い**
