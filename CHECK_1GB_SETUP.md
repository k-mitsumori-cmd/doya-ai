# 1GB動画処理の設定確認結果

## ✅ 確認完了項目

### 1. ファイルサイズ制限（1GB）
- **Next.js API**: `MAX_FILE_SIZE = 1024 * 1024 * 1024` (1GB) ✓
- **Cloud Runサービス**: `MAX_FILE_SIZE = 1024 * 1024 * 1024` (1GB) ✓
- 両方で1GB制限が設定されています

### 2. Cloud Runサービスのタイムアウト設定
- **タイムアウト**: 3600秒（1時間） ✓
- 大容量ファイルの処理に十分な時間が設定されています

### 3. GCSへのアップロード処理
- **アップロード先**: Google Cloud Storage (`doya-interview-storage`バケット) ✓
- **処理フロー**: 
  1. ユーザーがファイルをアップロード
  2. GCSに保存される
  3. GCS URI (`gs://...`) がデータベースに保存される
  4. 文字起こし時にGCS URIを使用して処理

### 4. MP4ファイルの処理
- **音声抽出**: FFmpegを使用してMP4からFLAC形式に変換 ✓
- **処理場所**: Cloud Runサービス内で実行 ✓
- **一時ファイル**: `/tmp`ディレクトリを使用（自動クリーンアップ） ✓

## 📋 処理フロー

```
1. ユーザーがMP4ファイルをアップロード
   ↓
2. Next.js APIがGCSにアップロード
   ↓
3. GCS URIがデータベースに保存
   ↓
4. 文字起こしリクエスト
   ↓
5. Next.js APIがCloud Runサービスを呼び出し
   ↓
6. Cloud RunサービスがGCSからMP4をダウンロード
   ↓
7. FFmpegで音声を抽出してFLACに変換
   ↓
8. FLACファイルをGCSにアップロード
   ↓
9. Google Cloud Speech-to-Text APIで文字起こし
   ↓
10. 結果を返す
```

**1GBの動画を処理できる設定になっています！**

- タイムアウト: 1時間 ✓
- GCSへのアップロード: 実装済み ✓
- MP4ファイルの音声抽出: FFmpegで実装済み ✓
