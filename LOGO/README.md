# ドヤロゴ（DOYA LOGO）

「サービス名」と「サービス内容」を入力するだけで、**日本っぽくてイケてるロゴ**を
**A/B/Cの3パターン** × **横長/正方形** × **SVG/PNG/JPEG** で一括生成し、
ロゴキット（ZIP）とガイドライン／生成理由までまとめて出力する仕組みです。

## できること

- **入力**: サービス名 / サービス内容 / 雰囲気 / 業界 / メイン色 / サブ色
- **ロゴ生成**:
  - Pattern A: 王道・汎用
  - Pattern B: 攻めた / 個性的
  - Pattern C: ミニマル / 長期運用向け
- **各パターンの出力**:
  - 横長ロゴ / 正方形ロゴ（アイコン）
  - SVG（必須）/ PNG（透過）/ JPEG（背景あり）
- **自動生成ドキュメント**:
  - カラーパレット（HEX/RGB/用途）
  - ロゴ使用ガイドライン（余白規定、最小サイズ、OK/NG、背景別推奨など）
  - 生成理由（Explainability）
- **ロゴキット（ZIP）**:
  - 全ロゴデータ + パレット資料 + ガイドライン + 生成理由

## 追加の独自機能（最低3つ以上）

- **ダークモード用ロゴ**自動生成
- **モノクロ版 / 反転版** 自動生成
- **Figma即貼り用SVG最適化**（軽量化・余計なメタ除去）
- **成長ストーリー**（拡張時の運用指針）自動生成
- **社内共有用「ロゴ説明一文」** 自動生成
- **商標チェックの注意コメント**自動生成（※法的判断ではなく注意喚起）

## 実行方法（Next.js 経由）

`doya-ai` の Next.js API から呼び出します。

- `POST /api/logo/generate`:
  - `returnMode: "zip"` で ZIP を返却
  - `returnMode: "json"` でプレビュー用 JSON を返却

## 出力ディレクトリ

生成物はデフォルトで `LOGO/outputs/<service-slug>/` に出力します（`.gitignore` 済み）。
本番やサーバレス環境では `DOYA_LOGO_OUTPUT_DIR` を `/tmp` などに切り替え可能です。
