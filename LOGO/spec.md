# ドヤロゴ（DOYA LOGO）仕様

## 目的

入力（サービス名・内容など）から、**日本市場で信頼されやすい、余白設計の効いたロゴ**を
**3パターン（A/B/C）** で同時生成し、ロゴキット（ZIP）として一括出力する。

## 入力（UI / API 共通）

- **serviceName**（必須）: サービス名
- **serviceDescription**（必須）: サービス内容（1〜3文推奨）
- **mood**（選択式）:
  - `japanese_modern`（日本的モダン）
  - `wa_tech`（和×テック）
  - `minimal`（ミニマル）
  - `bold`（力強い）
  - `startup`（スタートアップ感）
- **industry**（選択式）:
  - `saas` / `hr` / `ai` / `marketing` / `fintech` / `other`
- **mainColor**（任意）: HEX（`#RRGGBB`）
- **subColor**（任意）: HEX（`#RRGGBB`）

## 日本っぽさの定義（必須思想）

- 漢字・ひらがな・カタカナの「文字組」思想（文字が主役になれる設計）
- 余白設計（詰め込みすぎない）
- 極端に欧米寄りにしない（過度なセリフ/筆記体/派手な装飾を避ける）
- 無駄なグラデーションは使わない（基本フラット）
- 信頼感・長期運用（トレンド耐性のある骨格）

参考思想（直接コピー禁止）: Sansan / freee / SmartHR / LINE / メルカリ / Wantedly

## 生成仕様（最重要）

### 3パターン必須

- **Pattern A（王道・汎用）**
  - 読みやすさ・展開しやすさ・失敗しない構成
- **Pattern B（攻めた/個性的）**
  - 造形にフック、和の記号性（ハンコ/角印/紋章の抽象）
- **Pattern C（ミニマル/長期運用）**
  - 最小要素、縮小耐性、アイコン運用前提

### 各パターンで必ず生成

- **横長ロゴ**（横並び）
- **正方形ロゴ**（アイコン/SNS用）
- **ファイル形式**
  - SVG（必須）
  - PNG（透過）
  - JPEG（背景あり）

## カラーパレット自動生成

各パターンごとに以下を生成し、**HEX / RGB / 使用用途**を明示する。

- primary / secondary / accent
- grayscale（複数段階）
- usage（背景/テキスト/CTA など）

## ロゴキット（ZIP）

ZIPに含めるもの:

- 全ロゴデータ（SVG/PNG/JPEG）
- カラーパレット資料（`palette.json` + `palette.md`）
- ロゴ使用ガイドライン（`guideline.md`）
- 生成理由（`reasons.md`）
- 追加機能の成果物（例: `dark/mono/invert` や `mockups/`）

## ガイドライン自動生成（必須項目）

- ロゴの基本コンセプト
- 生成背景・思想
- 最小使用サイズ
- 余白規定（クリアスペース）
- OK例 / NG例（テキスト説明でOK）
- 背景色別の推奨ロゴ
- SNS / Web / 資料での使い分け

## Explainability（必須）

各パターンごとに:

- なぜこの形か
- なぜこの色か
- 内容とどう紐づくか
- 長期運用で強い理由

## 独自性・便益（最低3つ以上）

実装対象（本実装では複数同時に提供）:

- 成長ストーリー
- 利用サンプル（スライド/LP/名刺モック）
- Figma即貼りSVG最適化
- ダークモード
- モノクロ/反転
- 商標注意コメント
- ロゴ説明一文
