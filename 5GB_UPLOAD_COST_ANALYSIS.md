# 5GB動画アップロード対応のコスト分析

## 想定シナリオ

- 動画ファイルサイズ: 5GB
- 動画の長さ: 約2時間（一般的な動画のビットレートを想定）
- 音声抽出後のFLACファイルサイズ: 約500MB（動画の約10%）
- 処理時間: 約3時間（FFmpeg処理 + 文字起こし）

## コスト内訳

### 1. Google Cloud Run（文字起こし処理）

#### 現在の設定（1GB対応）
- メモリ: 2Gi
- CPU: 2コア
- タイムアウト: 3600秒（1時間）
- 処理時間: 約30分（1GBファイルの場合）

**コスト（1GBファイル）:**
- メモリ: 2Gi × $0.0000025/GB秒 × 1800秒 = **$0.009**（約1.4円）
- CPU: 2コア × $0.00002400/vCPU秒 × 1800秒 = **$0.0864**（約13円）
- **合計: 約$0.095（約14円）**

#### 5GB対応後の設定
- メモリ: 4Gi
- CPU: 2コア
- タイムアウト: 3600秒（1時間）
- 処理時間: 約3時間（5GBファイルの場合）

**コスト（5GBファイル）:**
- メモリ: 4Gi × $0.0000025/GB秒 × 10800秒（3時間） = **$0.108**（約16円）
- CPU: 2コア × $0.00002400/vCPU秒 × 10800秒 = **$0.5184**（約78円）
- **合計: 約$0.63（約94円）**

**注意**: Cloud Runのタイムアウトは最大3600秒（1時間）のため、3時間の処理には複数のリクエストまたは非同期処理が必要です。

### 2. Google Cloud Storage（ストレージ）

#### ストレージコスト
- Standard Storage: $0.020/GB/月
- 5GBファイル: 5GB × $0.020 = **$0.10/月**（約15円/月）

#### データ転送コスト
- アップロード（GCSへの書き込み）: 無料
- ダウンロード（Cloud Runから読み取り）: 無料（同一リージョン内）
- 外部へのダウンロード: $0.12/GB（最初の10TB/月）

**合計（ストレージのみ）: 約$0.10/月（約15円/月）**

### 3. Google Cloud Speech-to-Text API

#### 料金（2024年時点）
- 音声認識（標準モデル）: $0.006/15秒
- 音声認識（最新モデル）: $0.009/15秒（現在使用中: `latest_long`）

#### 5GB動画（約2時間の音声）の場合
- 音声の長さ: 約120分（7200秒）
- 15秒単位: 7200秒 ÷ 15 = 480単位
- コスト: 480 × $0.009 = **$4.32**（約648円）

**注意**: 実際の音声の長さは動画の長さと同じです。5GBの動画が2時間の場合、音声も2時間です。

### 4. FFmpeg処理（Cloud Run内で実行）

FFmpeg処理はCloud Runのリソースを使用するため、上記のCloud Runコストに含まれます。

## コスト削減のヒント

### 1. ストレージのライフサイクル管理
- 3ヶ月経過したファイルを自動削除（既に実装済み）
- 古いファイルを削除することでストレージコストを削減

### 2. 処理の最適化
- 不要な動画ファイルは音声抽出後に削除
- 音声ファイルのみを保存することでストレージコストを削減

### 3. バッチ処理
- 複数のファイルをまとめて処理することで、Cloud Runの起動コストを削減

### 4. リージョンの選択
- 同一リージョン内でのデータ転送は無料
- アジア太平洋リージョン（asia-northeast1）を使用することで転送コストを削減

## 現在の1GB対応との比較

### 1GBファイル（約30分の音声）の場合
- Cloud Run: 約$0.095（約14円）
- Speech-to-Text API: 約$0.54（約81円）
- **合計: 約$0.64（約96円）**

### 5GBファイル（約2時間の音声）の場合
- Cloud Run: 約$0.63（約94円）
- Speech-to-Text API: 約$4.32（約648円）
- **合計: 約$4.95（約742円）**

**5GBは1GBの約7.7倍のコスト**

## 注意事項

1. **Cloud Runのタイムアウト制限**
   - 最大3600秒（1時間）のため、3時間の処理には非同期処理が必要
   - または、処理を分割して複数のリクエストに分ける必要がある

2. **実際のコストは変動する**
   - 動画の長さ、音声の品質、処理時間によって変動
   - 上記は概算です

3. **無料枠**
   - Google Cloudには無料枠がありますが、5GBファイルの処理では無料枠を超える可能性が高い

## 推奨事項

1. **段階的な導入**
   - まず2GB、3GBなど段階的に上限を上げてテスト
   - 実際のコストを確認してから5GBに拡張

2. **ユーザーへの通知**
   - 大容量ファイルの処理には時間とコストがかかることを通知
   - 処理時間の目安を表示

3. **コスト監視**
   - Google Cloud Consoleでコストを監視
   - 予算アラートを設定
