# ユーザーストーリー

## ユーザーストーリー1: マーケティング担当者として事例記事を作成したい

**ユーザー**: BtoBマーケティング担当者（田中さん、30代）

**背景**: 
- 月に10本以上の事例記事を作成する必要がある
- インタビュー動画の文字起こしに時間がかかる
- 記事執筆にも時間がかかり、他の業務に影響が出ている

**ストーリー**:
```
インタビュー動画の音声ファイルをアップロードすると、
自動で文字起こしが行われ、
その内容を元に事例記事が生成されることで、
事例制作の工数を大幅に削減したい。
```

**受け入れ基準**:
- [ ] 音声ファイル（MP3/WAV）をアップロードできる
- [ ] 文字起こし結果が表示される
- [ ] 事例記事が自動生成される
- [ ] 生成された記事をコピー・ダウンロードできる
- [ ] 記事は「課題→解決策→効果」の構成になっている

**価値**: 事例制作時間を70%削減

---

## ユーザーストーリー2: 採用広報担当者として採用事例を素早く作成したい

**ユーザー**: 採用広報担当者（佐藤さん、20代）

**背景**:
- 採用動画を複数作成している
- 各動画から採用事例記事を作成する必要がある
- 動画編集と記事作成の両方を担当しており、時間が足りない

**ストーリー**:
```
採用インタビューの音声をアップロードすると、
採用事例記事が自動生成され、
すぐに採用サイトに掲載できる形式で出力されることで、
採用コンテンツ制作のスピードを上げたい。
```

**受け入れ基準**:
- [ ] 音声ファイルを簡単にアップロードできる
- [ ] 処理が完了するまで待機時間が明確に表示される
- [ ] 生成された記事が採用サイトに掲載できる形式である
- [ ] 記事の内容が読みやすく、実務で使える品質である

**価値**: 採用コンテンツ制作時間を50%削減

---

## ユーザーストーリー3: 制作担当者として複数の事例を一括処理したい

**ユーザー**: 制作担当者（鈴木さん、40代）

**背景**:
- 複数のクライアントの事例記事を同時に制作している
- 各クライアントから音声ファイルが送られてくる
- 効率的に処理したいが、現在は1つずつ手作業で処理している

**ストーリー**:
```
複数の音声ファイルを一度にアップロードすると、
それぞれの文字起こしと記事生成が自動で行われ、
一括でダウンロードできることで、
複数の事例を効率的に処理したい。
```

**受け入れ基準**:
- [ ] 複数ファイルを同時にアップロードできる（将来実装）
- [ ] 各ファイルの処理状況が個別に表示される
- [ ] 処理完了後、一括でダウンロードできる
- [ ] 各記事が適切に区別されて保存される

**価値**: 複数案件の処理時間を60%削減

## ユーザーストーリー4: マーケティングマネージャーとして品質を確認したい

**ユーザー**: マーケティングマネージャー（山田さん、40代）

**背景**:
- チームメンバーが作成した事例記事の品質を確認する必要がある
- 記事の内容を編集・修正する必要がある
- 過去の記事を参照したいことがある

**ストーリー**:
```
生成された記事を確認し、
必要に応じて編集・修正ができ、
過去の記事履歴を参照できることで、
記事の品質管理を効率的に行いたい。
```

**受け入れ基準**:
- [ ] 生成された記事を確認できる
- [ ] 記事を編集・修正できる（将来実装）
- [ ] 過去の記事履歴を参照できる（将来実装）
- [ ] 記事のバージョン管理ができる（将来実装）

**価値**: 品質管理時間を40%削減

## ユーザーストーリー5: 新規ユーザーとして簡単に使い始めたい

**ユーザー**: 新規ユーザー（初めて使う人）

**背景**:
- ツールの使い方がわからない
- どのような形式のファイルが使えるかわからない
- エラーが発生した時の対処法がわからない

**ストーリー**:
```
直感的なUIで操作でき、
エラーが発生した場合も適切なメッセージが表示され、
使い方を理解できることで、
すぐに使い始めたい。
```

**受け入れ基準**:
- [ ] 操作が直感的でわかりやすい
- [ ] エラーメッセージが明確で対処法がわかる
- [ ] 対応ファイル形式が明確に表示されている
- [ ] 処理の進捗がわかりやすく表示される

**価値**: 学習コストの削減、ユーザー満足度の向上

## ユーザーストーリー6: モバイルユーザーとしてスマホで使いたい

**ユーザー**: モバイルユーザー（外出先で作業する人）

**背景**:
- 外出先でインタビューを録音することがある
- すぐに記事を作成したい
- スマホからでも使いやすいツールが必要

**ストーリー**:
```
スマホからでも音声ファイルをアップロードし、
事例記事を生成・確認できることで、
外出先でも作業を進めたい。
```

**受け入れ基準**:
- [ ] スマホの画面サイズで適切に表示される
- [ ] タッチ操作で使いやすい
- [ ] ファイルアップロードがスマホからでも簡単
- [ ] 生成された記事がスマホでも読みやすい

**価値**: 作業場所の制約をなくす、柔軟な働き方の実現

## 成功指標（KPI）

### 定量指標
- **処理時間**: 1件あたりの処理時間を5分以内に
- **エラー率**: エラー発生率を5%以下に
- **ユーザー満足度**: ユーザー満足度スコアを4.0/5.0以上に

### 定性指標
- **使いやすさ**: 新規ユーザーが迷わず使える
- **記事品質**: 生成された記事が実務で使える品質
- **信頼性**: 安定して動作する
