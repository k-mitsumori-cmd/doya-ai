## 1. ドヤ営業管理とは（完成イメージの共有）

ドヤ営業管理は中小企業（5〜100名）向けの**SFA（営業支援）/ 営業管理ツール**です。「Salesforceは重すぎる」層に向けた、シンプル・かわいい・安い・**AI標準搭載**のツール。主な機能：

- **取引先・担当者管理**（会社と人を紐づけ、タイムラインで履歴を一覧）
- **リード管理**（見込み客を貯め、有望ならワンクリックで取引先＋商談に転換）
- **商談パイプライン（カンバン）**＝主役画面。ステージをドラッグ＆ドロップで移動
- **活動・タスク管理**（電話/訪問/メール/メモ、期日付きToDo）
- **ダッシュボード・売上予測・レポート**
- **AI機能**（リードスコアリング・次アクション提案・議事録・メール文面・自然言語検索）
- **権限管理**（オーナー/管理者/マネージャー/メンバー）と**組織単位のデータ分離**
- **課金**（無料プラン＋有料プラン、全サービス共通の統一プラン）

> 💡ポイント：最初から全部を狙わず、**「まず商談パイプライン（カンバン）」を完成**させ、
> そこにリード・活動・AI…と足していくと崩れません。営業ツールは**カンバンが土台**です。

## 2. 「このように作ればうまくいく」― 成功の設計指針

営業管理ツールならではの、最初に握っておくべき指針です。**ここを外すと後で必ず作り直しになります。**

1. **「会社・人・商談・活動」の関係を最初に固める**
   取引先（会社）に担当者（人）が紐づき、商談（案件）がぶら下がり、活動が記録される。
   この**データの骨格**を最初に設計させる。後から関係を変えると全画面に波及します。
   → 依頼：「まず**データの関係図**（取引先・担当者・リード・商談・活動・タスク）を設計して」

2. **組織ごとにデータを必ず分離する（マルチテナント）**
   A社のデータをB社が見たら一発アウト。全データに「**どの組織のものか**」を持たせ、
   **全ての取得・更新で組織IDの一致を必ず確認**する。
   → 依頼：「全データを**組織IDで分離**し、他組織のデータは絶対に見えない／触れないようにして」

3. **役割（権限）を最初に決める**
   メンバーは自分の担当だけ、マネージャーはチーム全体、管理者は組織設定…と段階を決める。
   特に「**オーナー権限は他人に勝手に渡せない**」等の昇格ガードを最初から。
   → 依頼：「役割は オーナー>管理者>マネージャー>メンバー。**権限の昇格防止**も入れて」

4. **入力負荷を減らすことが価値の本体**
   営業ツールは「入力が面倒だと使われない」。**3クリックで1件登録**、空画面にはサンプルとガイド。
   → 依頼：「**3クリック以内で1件登録**できるUIに。新規組織には**サンプルデータ**を自動投入して」

5. **AIで“入力の肩代わり”をする**
   次アクション提案・議事録・メール文面・スコアリング。**全部入力を減らす方向**に効かせる。
   → 依頼：「停滞商談に**次の一手＋期日付きタスク候補**を出すAIを。チェックして一括追加できるUIで」

6. **招待・組織の初期セットアップを自動化する**
   初回ログインで組織を自動作成し、既定の営業ステージとサンプルを投入。招待はメールリンク＋有効期限。
   → 依頼：「初回ログインで**組織自動作成＋既定パイプライン＋サンプル投入**。招待はトークン＋48時間期限で」

## 6. ④ テスト・不具合報告のしかた（品質を決める）

営業ツールは「**権限**」と「**データの紐づき**」が壊れやすい所です。複数ユーザー・複数組織で試すのが肝。

### 効く不具合報告の型

```
【症状】[誰の権限で／どの画面で] [何が起きたか]
【再現手順】1. … 2. … 3. …
【期待】本来こうなってほしい
【手がかり】（あれば）エラー文・HTTPステータス(404/405/500)・通信(API)の状態
```

営業ツールで「効く」報告の例：

- 「**別組織のメンバーで**ログインしたら、他社の商談が見えてしまった」→ 組織分離の漏れ（重大）
- 「商談カードを動かすと**一瞬動くが元に戻る**。`PATCH /api/.../deals/{id}` が500」→ 保存失敗
- 「リードを取引先に転換したのに**商談が作られない**」→ 転換処理の抜け

> 💡コツ：**「どの役割（権限）で操作したか」**を必ず書く。権限バグは役割を変えると再現します。

## 7. ⑤ ローンチ前チェックリスト

- [ ] 主要導線が全部動く（取引先作成→商談作成→カンバン移動→詳細編集→削除）
- [ ] **新規組織直後**（データ空）でも画面が壊れない（サンプル投入・空配列対応）
- [ ] 一覧⇄詳細⇄編集の**リンク先が全部存在**する（「ページが見つかりません」が出ない）
- [ ] **組織分離**：別組織のデータが一切見えない／編集できない（複数組織で実機確認）
- [ ] **権限**：メンバーは他人の担当を編集できない／オーナー権限は勝手に渡せない
- [ ] カンバンの**ドラッグ＆ドロップ**で移動・保存が効く（タッチ用フォールバックも）
- [ ] AI（次アクション・議事録・メール文面）が動き、結果を**保存／タスク化**できる
- [ ] エラー時に**日本語**で分かるメッセージが出る
- [ ] 料金表示が**全画面で一致**（統一プラン、価格は1か所で管理）
- [ ] 招待メール：**有効期限切れ**で正しく弾かれる
- [ ] CSV入出力が動く（取引先・リード・商談）
- [ ] 公開ステータスを「準備中」から「公開（active）」に変更
- [ ] 不要な機能は**消すのではなく非表示**にして、後で戻せるように

## 8. つまずきやすいポイント（ドヤ営業管理 実例つき）

1. **組織分離の漏れ（最重要・事故ると致命的）**
   IDを指定して取得する処理で、組織IDの一致確認を忘れ、他組織のデータが取れてしまう。
   → 「**ID指定の後に必ず組織IDの一致を確認**して。不一致は『見つかりません』を返して」

2. **新規データのnullで画面クラッシュ**
   作成直後は紐づくリスト（活動・タスク等）が空(null)で、`.length`参照でエラー。
   → 「新規作成直後でも壊れないように、配列は**空配列で初期化**して」

3. **ルーティング／APIメソッドの抜け（404/405）**
   一覧から詳細・編集へ飛ぶのに、そのページやAPI（GET/PATCH）が無い。
   → 「一覧・詳細・編集・各APIの**メソッド（GET/POST/PATCH/DELETE）が全部揃っているか**確認して」

4. **権限の作り込み不足**
   メンバーが他人の担当データを編集できてしまう／権限を勝手に上げられる。
   → 「**自分の担当だけ編集可、昇格は上位者のみ**。役割の判定を1か所にまとめて」

5. **リード転換の取りこぼし**
   リードを「取引先＋担当者＋商談」に変換する処理で、一部しか作られない。
   → 「転換時に**取引先・担当者・商談をまとめて生成**し、元リードは『転換済』にして」

6. **価格・設定の二重定義**
   料金を複数ファイルに直書きして画面ごとに金額がバラバラ。
   → 「価格は**1か所（単一ソース）**で管理し、全画面がそこを参照する形にして」

7. **並行で複数AIセッションを同じプロジェクトに走らせない**
   ファイルの取り合いで変更が消えます。**1プロジェクト＝1セッション**。

8. **削除より「非表示」**
   使わない機能・画面はコードごと消さず非表示に。要件が戻ったときの手戻りが小さい。

## 10. 技術メモ（依頼相手＝開発担当者向けの参考）

ドヤ営業管理の構成（同種ツールの“型”として）：

- フロント：Next.js（App Router）＋ React ＋ TypeScript、Tailwind CSS、Framer Motion
- データ：PostgreSQL ＋ Prisma。**`organizationId` 列で組織ごとにデータ分離**（マルチテナント）
- 中核モデル：取引先 / 担当者 / リード / 商談 / 商談ステージ / 活動 / タスク（会社→人→商談→活動の関係）
- 認証：NextAuth（Googleログイン）／役割（オーナー・管理者・マネージャー・メンバー）
- セキュリティ：全APIで認証必須、**ID指定の後に組織IDの一致を確認**（他組織アクセス防止）、役割/ステータスは**ホワイトリスト検証**
- カンバン：ドラッグ＆ドロップでステージ移動（楽観更新＋保存）。タッチ用に選択UIを併設
- AI：リードスコアリング・次アクション提案・議事録・メール文面・自然言語検索（テキストLLM）
- 課金：Stripe（**全サービス共通の統一プラン**、プラン判定はユーザーのプラン列1か所）
- 通知：共通トースト（`react-hot-toast`）、デザインは共通サイドバー／テーマに統一

> このメモは「同じ作法で増築する」ための地図です。営業ツールは特に
> **組織分離・権限・データの紐づき**を最初に固めておくと、事故らず長く保守できます。
