# データベースに「記事作成っぽい画像」を追加する方法

## 現状の問題点

現在の画像生成プロンプトは「フラットなベクターアイコン」という抽象的な方向で、記事作成のコンテキストが薄い。

## 改善提案

### 1. プロンプトを「記事作成」テーマに寄せる

以下のようなキーワードを追加：
- ライティング、記事作成、コンテンツ作成
- ノートパソコン、キーボード、ペン、メモ帳
- SEO、マーケティング、コンテンツマーケティング
- 編集、執筆、投稿

### 2. カテゴリごとのバリエーションを増やす

#### 「記事の種類」カテゴリ
- 比較記事：天秤、チャート、並列アイコン
- HowTo記事：手順/ステップ、リスト、チェックマーク
- 解説記事：本、知恵袋、解説アイコン
- 事例記事：成功/完了、グラフ上昇、結果アイコン

#### 「記事の方向性」カテゴリ
- ターゲット読者に向けた矢印
- コンテンツマーケティングのフローチャート
- SEO最適化のターゲットと矢印

#### 「キーワード」カテゴリ
- 検索バー + タグ
- キーワードマップ/マインドマップ
- SEOキーワードのハッシュタグ

#### 「ターゲット読者」カテゴリ
- ペルソナカード、ユーザーアイコン
- 読者層のシルエット（3-4人）

#### 「記事の長さ」カテゴリ
- 文字数/ワードカウントのメーター
- 進捗バー、完成度ゲージ
- ページ/章のアイコン

### 4. 実装方法

#### オプション1: 既存プロンプトを改善（推奨）
`/api/swipe/question-images/generate`の`categoryPrompts`を上記に更新。

#### オプション2: カテゴリ別生成オプションを追加
生成UIでカテゴリを選択できるようにする。

#### オプション3: バッチ生成
各カテゴリごとに10-20枚ずつ生成し、バリエーションを増やす。

### 5. 推奨アクション

1. **既存プロンプトの改善**（上記プロンプト例を反映）
2. **生成枚数の増加**（デフォルト30枚→60-90枚）
3. **カテゴリごとのバリエーション確保**（各カテゴリ10-15枚程度）

### 6. 生成方法

既存の生成ページ `/seo/test/generate-question-images` を使用：
- 生成枚数を60-90に設定
- 「画像を生成する」ボタンをクリック
- 1枚あたり約2秒かかるため、完了まで待機

### 7. 注意事項

- 画像内にテキスト/文字を含めない（Geminiに明示的に指定）
- フラットなベクター/イラストのみ（写真風は避ける）
- 花・植物モチーフは避ける
- 背景は明るいグラデーション、被写体は中央、余白多め
