## 🚀 開発開始時の流れ

新サービス開発を開始する際は、以下の順番で進めてください：

### Step 1: 共通項目の確認

**👉 [development-guide.md](./development-guide.md#共通項目必須実装の確認) の「共通項目・必須実装の確認」を確認**

開発開始前に、全サービス共通で必須の項目を確認：
- サービス定義の必須フィールド
- ページ構成
- サイドバー・レイアウトの必須機能
- プラン管理の必須実装
- レスポンシブデザインの必須対応
- デザイン統一の必須準拠
- サービス分離の必須遵守

### Step 2: チェックリストの取得

**👉 [development-guide.md](./development-guide.md#新サービス開発-完全チェックリスト) の「新サービス開発 完全チェックリスト」をコピー**

実装中に使用するチェックリストです。10フェーズ、100項目以上のチェックリストを順番に確認しながら実装します。

### Step 3: 実装開始

**👉 [development-guide.md](./development-guide.md#新サービス追加手順) の「新サービス追加手順」を参照**

詳細な実装手順に従って実装を進めます。

## 📝 クイックチェックリスト（開発開始時）

開発を開始する前に、以下を確認：

- [ ] サービスIDが他のサービスと重複していない
- [ ] サービス定義の必須フィールドを確認した
- [ ] プランの `dailyLimit` が適切に設定されている
- [ ] ベータ版サービスの場合は `status: 'beta'` を設定することを理解した
- [ ] 既存のサイドバーパターンを参考にすることを理解した
- [ ] 既存のレイアウトパターンを参考にすることを理解した
- [ ] レスポンシブデザインを実装することを理解した
- [ ] プラン管理を実装することを理解した
- [ ] サービス分離ルールを遵守することを理解した

**詳細なチェックリスト**: [新サービス開発 完全チェックリスト](./development-guide.md#新サービス開発-完全チェックリスト)

## 原則

1. **サービス分離**: 他サービスのAPI/DB/課金に絶対に触れない
2. **共通部品活用**: ToolSwitcherMenu, 認証, 決済は共通
3. **デザイン統一**: CSS変数・Tailwindテーマを使用
4. **アニメーション統一**: Party Mode対応
5. **外部API活用**: SerpAPI, Sora APIなどの検索系APIを積極的に活用

## デプロイ

変更を本番環境に反映する方法：

- **詳細**: [development-guide.md](./development-guide.md#デプロイ手順) の「デプロイ手順」セクション
- **手順**: `git push origin main` でVercelが自動デプロイ
- **確認**: Vercelダッシュボードでビルドログを確認
