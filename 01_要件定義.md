# 音声アップロード型 事例記事自動生成アプリ 要件定義

## 1. アプリ概要

### 1.1 目的
インタビュー動画の音声ファイルをアップロードし、自動で文字起こしを行い、その内容を元に「事例記事」を自動生成するWebアプリケーション。

### 1.2 ターゲットユーザー
- BtoBマーケティング担当者
- 採用広報担当者
- コンテンツ制作担当者

### 1.3 解決する課題
- 事例制作の工数削減（動画撮影 → 音声 → 記事の自動化）
- 手動での文字起こし作業の削減
- 記事執筆の時間短縮

## 2. 機能要件

### 2.1 必須機能

#### 2.1.1 音声ファイルアップロード機能
- **対応形式**: MP3, WAV
- **最大ファイルサイズ**: 100MB（初期実装では制限なし、後で実装可能）
- **アップロード方法**: ドラッグ&ドロップ or ファイル選択
- **バリデーション**: ファイル形式チェック、サイズチェック

#### 2.1.2 文字起こし機能（疑似処理）
- アップロードされた音声ファイルを「文字起こし中」として処理
- ダミーデータで文字起こし結果を返却
- 後でWhisper APIやClaude APIに置き換え可能な構造

#### 2.1.3 事例記事自動生成機能
- 文字起こし結果を元に記事を自動生成
- **出力フォーマット**:
  - リード文（導入文）
  - 見出し（H2/H3）
  - 課題 → 解決 → 効果の構成
- 後でClaude APIやGPT APIに置き換え可能な構造

#### 2.1.4 結果表示機能
- 文字起こし結果の表示
- 生成された記事の表示
- 記事のコピー機能
- 記事のダウンロード機能（テキストファイル）

### 2.2 UI/UX要件

#### 2.2.1 ページ構成
1. **アップロード画面**
   - ファイル選択エリア
   - アップロードボタン
   - 進捗表示

2. **処理中画面**
   - ローディング表示
   - 処理ステップの表示（アップロード → 文字起こし → 記事生成）

3. **結果表示画面**
   - 文字起こし結果タブ
   - 生成記事タブ
   - コピー/ダウンロードボタン

#### 2.2.2 デザイン要件
- シンプルで実務レベルのデザイン
- レスポンシブ対応（PC/タブレット/スマホ）
- 直感的な操作フロー
- エラーハンドリングとメッセージ表示

### 2.3 技術要件

#### 2.3.1 フロントエンド
- HTML5
- CSS3（モダンなスタイリング）
- JavaScript（ES6+、純JSのみ、ライブラリ不要）
- ファイルAPI使用

#### 2.3.2 拡張性
- API連携用の関数を分離
- 疑似処理部分を後で置き換え可能な構造
- 設定ファイルや環境変数でAPIキーを管理できる設計

#### 2.3.3 Studio埋め込み対応
- iframeで埋め込める構造
- 外部からのアクセスを考慮した設計
- サイズ調整可能

## 3. データフロー

```
1. ユーザーが音声ファイルをアップロード
   ↓
2. ファイルを読み込み（FileReader API）
   ↓
3. 文字起こし処理を開始（疑似処理）
   - 現在: ダミーデータ生成
   - 将来: Whisper API呼び出し
   ↓
4. 文字起こし結果を取得
   ↓
5. 記事生成処理を開始（疑似処理）
   - 現在: テンプレートベースのダミー生成
   - 将来: Claude API/GPT API呼び出し
   ↓
6. 生成された記事を表示
   ↓
7. ユーザーがコピー/ダウンロード
```

## 4. 記事テンプレート構成

### 4.1 基本構成
```
【リード文】
（導入文：企業・人物の紹介、インタビューの背景）

【H2見出し：課題】
（企業が抱えていた課題の説明）

【H2見出し：解決策】
（導入したソリューションや取り組みの説明）

【H2見出し：効果・成果】
（導入後の効果や成果の説明）

【H2見出し：まとめ】
（まとめと今後の展望）
```

### 4.2 詳細見出し（H3）
各H2セクション内に、必要に応じてH3見出しを配置

## 5. 非機能要件

### 5.1 パフォーマンス
- ファイルアップロードの進捗表示
- 処理中の適切なフィードバック

### 5.2 セキュリティ
- クライアントサイドのみで動作（初期実装）
- ファイルはブラウザ内で処理（サーバーに送信しない）

### 5.3 互換性
- モダンブラウザ対応（Chrome, Firefox, Safari, Edge）
- モバイルブラウザ対応

## 6. 将来の拡張機能

### 6.1 API連携
- Whisper API（文字起こし）
- Claude API / GPT-4（記事生成）
- 音声ファイルのストレージ保存（S3等）

### 6.2 機能追加
- 複数ファイルの一括処理
- 記事テンプレートのカスタマイズ
- 編集機能（生成後の記事を手動編集）
- 履歴保存機能
